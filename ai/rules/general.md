# General rules

## Purpose
Define strict transversal constraints across the project.

## Scope
Applies to all code, documentation, and AI-generated artifacts.

## Core principles
- Consistency across the system.
- Explicitness over implicit behavior.
- Simplicity over unnecessary abstraction.

## Mandatory rules
- Project structure must be respected.
- All changes must update related documentation.
- No unused dependencies allowed.
- No unused code allowed.
- Follow existing naming and architectural conventions.
- Avoid premature optimization.
- Avoid over-engineering.
- Every module must have a clear responsibility.
- Cross-module behavior must be explicit.

## Additional considerations
Refer to:
- `ai/rules/clean-code.md`
- `ai/rules/architecture.md`

## Rule priority
General rules apply unless a more specific rule overrides them.
